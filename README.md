# Art by Natalia — сайт-портфолио художника и скульптора

Премиальный сайт-галерея для художника и скульптора **Натальи Калугиной (Шилко)**.
Next.js 14 (App Router) + TypeScript + Tailwind CSS + Framer Motion.

## Что внутри

- Светлая и тёмная темы с плавным переключателем (400–600 мс)
- Главная страница с анимированным Hero, живым золотым фоном из частиц
- Каталог с фильтрами (категория, цена, наличие, поиск, сортировка)
- Страница работы с галереей и кнопками WhatsApp / Instagram
- Страница «О художнике» с текстом и фактами
- Страница «Контакты» с кнопками (без формы)
- Админ-панель с защищённым входом по логину/паролю
- AI чат-помощник (Groq) со знаниями о художнике и каталоге
- Оригинальные переходы между страницами и премиальные hover-эффекты

## Быстрый старт

```bash
npm install
cp .env.example .env.local
# впишите свои значения в .env.local
npm run dev
```

Откройте http://localhost:3000

## Переменные окружения (`.env.local`)

| Переменная | Назначение |
|---|---|
| `ADMIN_LOGIN` | логин администратора (можно менять когда угодно) |
| `ADMIN_PASSWORD` | пароль администратора |
| `ADMIN_SESSION_SECRET` | случайная длинная строка для подписи сессии — сгенерируйте свою |
| `GROQ_API_KEY` | API-ключ Groq для чат-помощника |
| `GROQ_MODEL` | модель Groq, по умолчанию `llama-3.3-70b-versatile` |

Логин и пароль **нигде не хранятся в коде** — только в переменных окружения,
поэтому их можно поменять в один клик в настройках проекта на Vercel,
без изменения самого сайта. Постороннему человеку без .env-значений
попасть в `/admin/dashboard` невозможно — маршрут защищён серверной проверкой
(см. `middleware.ts`).

## Админ-панель

1. Откройте `/admin` и войдите по логину и паролю.
2. В разделе **«Работы»** добавляйте, редактируйте и удаляйте картины,
   скульптуры и сувениры, загружайте фотографии, указывайте размер, вес,
   материал, цену, статус и наличие.
3. В разделе **«Экспорт / импорт»** можно скачать резервную копию каталога
   в файл и загрузить её позже — это способ перенести данные между
   устройствами или сохранить бэкап.

**Важно про хранение данных:** каталог работ хранится в `localStorage`
браузера — это позволяет обойтись без базы данных и разворачивать проект
прямо на Vercel. Если нужно, чтобы каталог был общим для всех посетителей
сайта (а не только виден в браузере, где выполнялся вход), потребуется
подключить настоящую базу данных (например, Vercel Postgres, Supabase
или Vercel KV) — тогда есть смысл заменить `lib/admin-store.ts` на запросы
к API. Сейчас это не сделано, чтобы проект можно было развернуть без
дополнительной инфраструктуры.

## Подключение AI-чата (Groq)

Впишите ключ в `GROQ_API_KEY` в `.env.local` (для локальной разработки)
или в Environment Variables проекта на Vercel (для продакшена).
Ключ используется только на сервере (`app/api/chat/route.ts`) и никогда
не попадает в браузер.

## Деплой на Vercel

1. Залейте проект в GitHub.
2. Импортируйте репозиторий на vercel.com.
3. Добавьте переменные окружения из `.env.example` в настройках проекта.
4. Deploy.
